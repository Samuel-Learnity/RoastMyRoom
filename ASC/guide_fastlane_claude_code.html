Guide Pratique

App Store Rejection
→ Resubmission en 12 min

Comment automatiser ta pipeline iOS avec Fastlane + Claude Code pour ne plus jamais perdre une demi-journée sur un rejet.

01

Le Problème

Tu reçois le mail d'Apple. Rejection. Ton estomac se noue. Tu sais ce qui t'attend : lire la lettre, comprendre les violations, trouver les fichiers concernés, appliquer les corrections, rebuild, re-archive, re-upload, mettre à jour les métadonnées dans 4 langues, re-soumettre...

  • ✕ Trouver les bons fichiers dans un projet de 50+ fichiers Swift
  • ✕ Modifier le code, tester, s'assurer que rien d'autre n'est cassé
  • ✕ Mettre à jour les descriptions App Store en 4 langues (EN, FR, DE, ES)
  • ✕ Naviguer dans l'UI App Store Connect pour upload chaque fichier
  • ✕ Archive Xcode → organizer → upload → attendre le processing
  • ✕ Vérifier que les screenshots matchent toujours l'UI modifiée
  • ✕ Submit for review — en priant pour que ce soit le dernier round

Réalité : Chaque rejet coûte 1 à 2 heures de travail manuel. Multiplie par 2–3 itérations de review, et tu perds une journée entière. Sur un side project, c'est une semaine de motivation.

02

La Solution : 2 Piliers

⚙
Fastlane
Pipeline automatisé qui gère metadata, screenshots, build, upload et submission en une commande. Plus jamais toucher l'UI d'App Store Connect.
⚡
Claude Code
Assistant IA qui comprend ton codebase (via CLAUDE.md), identifie les fichiers à modifier, applique les corrections, et lance le pipeline pour toi.

La combinaison gagnante : Claude Code comprend le « quoi » (analyser la lettre de rejet, identifier les fichiers, écrire le fix). Fastlane gère le « comment » (build, upload, submit). Ensemble, ils réduisent une tâche de 2h à 12 minutes.

03

Architecture du Pipeline

Voici les lanes Fastlane utilisées dans le projet RoastMyRoom. Chaque lane est une commande unique qui orchestre plusieurs étapes.

1
fastlane bump
Incrémente le build number dans le xcodeproj. Obligatoire avant chaque upload.
2
fastlane metadata
Upload les métadonnées localisées (4 langues) via l'API Spaceship. Descriptions, keywords, subtitle, URLs… le tout lu depuis des fichiers .txt.
3
fastlane screenshots
Upload les screenshots localisés (générés par un script Python). Supprime les anciens, upload les nouveaux.
4
fastlane archive_upload
Archive Release via xcodebuild + export avec ExportOptions.plist. Le binaire est envoyé directement à ASC.
5
fastlane release
Tout en un : build + metadata + screenshots + submit for review. La lane nucléaire.
fastlane/Fastfile — extrait
lane :archive_upload do
  sh("xcodebuild -project ../RoastMyRoom.xcodeproj \
    -scheme RoastMyRoom -configuration Release \
    -archivePath ../build/RoastMyRoom.xcarchive \
    archive -quiet")
  sh("xcodebuild -exportArchive \
    -archivePath ../build/RoastMyRoom.xcarchive \
    -exportOptionsPlist ../ExportOptions.plist \
    -exportPath ../build/export -quiet")
end

lane :metadata do
  ensure_localizations  # crée les locales manquantes sur ASC
  # Upload via API Spaceship directe
  # (contourne les bugs de deliver)
  LOCALES.each do |locale|
    # Lit description.txt, keywords.txt, etc.
    # Update via l'API
  end
end
04

Cas Réel : Rejet → Resoumission

Voici exactement ce qui s'est passé sur RoastMyRoom. Deux violations, un fix de 12 minutes.

⚠ Lettre de rejet Apple
Guideline 5.1.1 Data Collection and Storage — Le prompt ATT (App Tracking Transparency) présentait un bouton « Maybe Later » permettant de différer la réponse, ce qui est interdit.
Guideline 3.1.2 Subscriptions — Lien vers le EULA/Conditions d'utilisation manquant dans les descriptions App Store.
↓
⚡ Claude Code analyse et corrige
0:00
Analyse de la lettre. Claude Code identifie les 2 guidelines violées et les fichiers impliqués.
1:30
Fix 1 — ATT. Suppression du bouton « Maybe Later » dans ATTPrePromptView.swift. Le prompt natif ATT se déclenche directement. Guideline 5.1.1 résolue.
4:00
Fix 2 — EULA. Ajout des liens EULA et Privacy Policy dans les 4 fichiers description.txt (EN, FR, DE, ES). Guideline 3.1.2 résolue.
6:00
Build + Tests. Compilation sans warning, tests passés. Rien de cassé.
7:00
Pipeline. fastlane bump → fastlane metadata → fastlane archive_upload
12:00
Done. Build uploadé, metadata à jour dans les 4 langues. Soumission pour review.
05

Setup Rapide

Tout ce dont tu as besoin pour reproduire cette pipeline sur ton projet.

1. Gemfile

source "https://rubygems.org"

gem "fastlane"
gem "xcodeproj"

Puis bundle install pour installer les dépendances.

2. Structure métadonnées

ASC/ ├── metadata/ │ ├── en-US/ │ │ ├── name.txt ← App name │ │ ├── subtitle.txt ← 30 chars max │ │ ├── description.txt ← Full description │ │ ├── keywords.txt ← Comma-separated, 100 chars │ │ ┪── promotional_text.txt ← Editable without review │ ├── fr-FR/ │ ├── de-DE/ │ ┪── es-ES/ ├── output/ ← Screenshots localisés ┪── review_notes.txt ← Notes for Apple reviewers

3. CLAUDE.md

Le fichier CLAUDE.md à la racine du projet est la clé. Il donne à Claude Code le contexte nécessaire :

# Contenu typique d'un CLAUDE.md

## Build
- Scheme : "MonApp"
- Device de test : iPhone 16 Pro

## Architecture
- MVVM, SwiftUI, Swift 6.1
- Services injectés via AppFactory

## Fastlane
- fastlane metadata — upload metadata 4 langues
- fastlane archive_upload — archive + upload ASC
- fastlane release — full pipeline

## Folder Structure
- ASC/metadata/{locale}/ — App Store metadata
- fastlane/Fastfile — Pipeline config

Sans ce fichier, Claude Code doit explorer le projet à chaque fois. Avec, il sait immédiatement où chercher et quoi faire.

4. ExportOptions.plist

Requis pour xcodebuild -exportArchive. Spécifie la méthode d'export (app-store), le team ID et le provisioning profile.

06

Checklist Avant Soumission

Les raisons de rejet les plus fréquentes, et comment les prévenir. Coche chaque point avant de lancer fastlane release.

  • ✓
    EULA / Privacy Policy — Liens présents dans la description App Store ET accessibles in-app (Settings ou Paywall). Guideline 3.1.2.
  • ✓
    ATT Prompt — Pas de bouton « Skip » ou « Maybe Later ». Le prompt système doit être l'unique interface. Guideline 5.1.1.
  • ✓
    Subscriptions — Titre, durée, prix, conditions d'annulation et liens vers EULA/Privacy visibles avant l'achat. Guideline 3.1.2.
  • ✓
    Privacy Manifest — PrivacyInfo.xcprivacy inclus. Déclare les APIs utilisées (UserDefaults, disk space, etc.) et les raisons. Requis depuis iOS 17.
  • ✓
    Screenshots — Correspondent à l'UI actuelle de l'app. Si tu as modifié une vue, regénère les screenshots.
  • ✓
    Review Notes — Instructions claires pour l'équipe de review (credentials de test si login requis, instructions de navigation).
  • ✓
    Build Number — Incrémenté par rapport au dernier upload. fastlane bump le fait automatiquement.
  • ✓
    Content Ratings — Questionnaire rempli dans ASC. Doit correspondre au contenu réel de l'app.
